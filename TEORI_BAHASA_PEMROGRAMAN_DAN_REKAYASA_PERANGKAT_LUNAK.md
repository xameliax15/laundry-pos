# Teori Penunjang: Bahasa Pemrograman dan Rekayasa Perangkat Lunak

## Teori Tentang Bahasa Pemrograman

Pengembangan aplikasi Laundry POS System ini menggunakan pendekatan modern dalam pemilihan dan pemanfaatan bahasa pemrograman yang dirancang khusus untuk membangun aplikasi cross-platform yang efisien dan maintainable. Bahasa pemrograman utama yang digunakan adalah **Dart**, yang merupakan bahasa pemrograman yang dikembangkan oleh Google secara khusus untuk framework Flutter. Dart dipilih karena memiliki karakteristik yang sangat sesuai dengan kebutuhan pengembangan aplikasi modern, yaitu type-safe yang memastikan keamanan tipe data pada saat compile-time, null safety yang mencegah kesalahan null pointer exception yang sering terjadi dalam pengembangan aplikasi, serta dukungan native untuk asynchronous programming melalui fitur async/await yang memungkinkan penanganan operasi I/O seperti komunikasi dengan database dan API tanpa memblokir thread utama aplikasi.

Dalam konteks aplikasi Laundry POS System ini, Dart digunakan untuk menulis seluruh logika bisnis aplikasi, mulai dari manajemen transaksi, pengelolaan pelanggan, autentikasi pengguna, hingga komunikasi dengan backend Supabase. Karakteristik type-safe dari Dart sangat membantu dalam pengembangan aplikasi yang kompleks seperti sistem POS ini, karena kompiler Dart dapat mendeteksi kesalahan tipe data sebelum aplikasi dijalankan, sehingga mengurangi bug yang mungkin terjadi pada production. Fitur null safety yang diperkenalkan dalam Dart 2.12 memungkinkan developer untuk secara eksplisit mendeklarasikan apakah suatu variabel dapat bernilai null atau tidak, yang sangat penting dalam aplikasi yang banyak berinteraksi dengan database dan API eksternal dimana data yang diterima mungkin tidak selalu lengkap atau konsisten.

Aspek asynchronous programming dalam Dart memainkan peran krusial dalam aplikasi ini, terutama dalam operasi-operasi yang memerlukan komunikasi dengan Supabase sebagai backend service. Setiap operasi seperti mengambil data transaksi, menyimpan data pelanggan baru, atau mengunggah foto laundry ke Supabase Storage, semuanya dilakukan secara asynchronous menggunakan Future dan async/await pattern. Pendekatan ini memastikan bahwa antarmuka pengguna tetap responsif selama operasi jaringan berlangsung, memberikan pengalaman pengguna yang lebih baik dibandingkan dengan aplikasi yang melakukan operasi blocking. Dalam implementasinya, aplikasi menggunakan FutureBuilder widget dari Flutter yang secara otomatis menangani state loading, success, dan error dari operasi asynchronous, sehingga memudahkan developer dalam membangun UI yang responsif.

Selain Dart sebagai bahasa pemrograman utama, aplikasi ini juga memanfaatkan teknologi web rendering melalui HTML, CSS, dan JavaScript yang dihasilkan oleh Flutter ketika aplikasi dikompilasi untuk platform web. Flutter menggunakan skema kompilasi ke JavaScript melalui Dart-to-JavaScript compiler yang memungkinkan aplikasi yang ditulis dengan Dart dapat berjalan di browser web modern. Meskipun developer tidak menulis HTML, CSS, atau JavaScript secara langsung, pemahaman tentang bagaimana Flutter mengkonversi widget tree menjadi DOM elements dan bagaimana styling system Flutter bekerja sangat membantu dalam optimasi performa aplikasi web. Aplikasi Laundry POS System ini dirancang untuk dapat berjalan di berbagai platform termasuk web browser, sehingga pemahaman tentang web rendering menjadi penting dalam memastikan aplikasi dapat diakses melalui berbagai browser dengan performa yang optimal.

Pemahaman tentang bahasa pemrograman dalam konteks aplikasi ini diperoleh melalui berbagai sumber pembelajaran, termasuk Studi Independen yang memberikan dasar-dasar pemrograman Dart dan Flutter, serta pembelajaran mandiri melalui dokumentasi resmi, tutorial online, dan praktik langsung dalam pengembangan aplikasi. Pengalaman langsung dalam mengimplementasikan fitur-fitur kompleks seperti Finite State Machine untuk manajemen status transaksi, integrasi dengan Supabase untuk backend operations, dan implementasi offline-first architecture dengan SQLite, memberikan pemahaman mendalam tentang bagaimana karakteristik bahasa pemrograman Dart dapat dimanfaatkan secara optimal dalam pengembangan aplikasi enterprise-level.

## Teori Rekayasa Perangkat Lunak

Rekayasa perangkat lunak yang diterapkan dalam pengembangan aplikasi Laundry POS System ini mengaplikasikan prinsip-prinsip fundamental rekayasa perangkat lunak untuk merancang, mengembangkan, memelihara, dan mengelola perangkat lunak secara sistematis dan terstruktur. Tujuan utama dari penerapan teori rekayasa perangkat lunak ini adalah menghasilkan perangkat lunak yang berkualitas tinggi, dapat diandalkan, mudah dirawat, dan memenuhi kebutuhan pengguna dengan efektif. Dalam konteks aplikasi Laundry POS System, teori rekayasa perangkat lunak diterapkan melalui berbagai aspek mulai dari arsitektur aplikasi, metodologi pengembangan, manajemen kualitas kode, hingga proses deployment dan maintenance.

Arsitektur aplikasi ini mengikuti prinsip **layered architecture** atau arsitektur berlapis yang memisahkan concern aplikasi ke dalam beberapa layer yang memiliki tanggung jawab spesifik. Layer pertama adalah **presentation layer** yang terdiri dari berbagai halaman (pages) dan widget yang bertanggung jawab untuk menampilkan antarmuka pengguna dan menangani interaksi pengguna. Layer kedua adalah **business logic layer** yang diimplementasikan melalui service classes seperti TransaksiService, CustomerService, AuthService, dan FMSService yang berisi logika bisnis aplikasi. Layer ketiga adalah **data access layer** yang terdiri dari DatabaseHelper untuk SQLite local database dan integrasi dengan Supabase untuk remote database operations. Pemisahan ini memungkinkan setiap layer dapat dikembangkan, diuji, dan dimodifikasi secara independen tanpa mempengaruhi layer lainnya, sehingga meningkatkan maintainability dan testability aplikasi.

Prinsip **separation of concerns** diterapkan secara konsisten dalam struktur kode aplikasi, dimana setiap modul memiliki tanggung jawab yang jelas dan terbatas. Sebagai contoh, TransactionFSM (Finite State Machine) yang digunakan untuk mengelola alur status transaksi dipisahkan dari logika bisnis lainnya, sehingga perubahan pada logika state management tidak akan mempengaruhi bagian lain dari aplikasi. Demikian pula, theme system yang terdiri dari AppColors, AppTextStyles, dan AppDimensions dipisahkan dari komponen UI, sehingga perubahan desain dapat dilakukan tanpa memodifikasi logika aplikasi. Pendekatan ini memungkinkan aplikasi untuk berkembang secara modular, dimana fitur-fitur baru dapat ditambahkan tanpa mengganggu fungsionalitas yang sudah ada.

Metodologi pengembangan yang diterapkan dalam proyek ini mengikuti pendekatan **iterative development** yang memungkinkan pengembangan fitur secara bertahap dengan siklus feedback yang cepat. Setiap fitur dikembangkan melalui beberapa iterasi, mulai dari implementasi dasar, pengujian, perbaikan bug, hingga optimasi performa. Pendekatan ini memungkinkan aplikasi untuk terus berkembang sesuai dengan kebutuhan yang muncul selama proses pengembangan, tanpa harus menunggu seluruh fitur selesai sebelum dapat digunakan. Dalam konteks aplikasi Laundry POS System, fitur-fitur seperti dashboard kasir, manajemen transaksi, dan tracking kurir dikembangkan secara bertahap, dimana setiap tahap dapat diuji dan dievaluasi sebelum melanjutkan ke tahap berikutnya.

Prinsip **DRY (Don't Repeat Yourself)** diterapkan dalam pengembangan aplikasi melalui penggunaan widget yang dapat digunakan kembali (reusable widgets) dan service classes yang dapat dipanggil dari berbagai bagian aplikasi. Sebagai contoh, komponen seperti KasirTransactionCard, KasirStatsCard, dan EmptyStateWidget dirancang sebagai widget yang dapat digunakan kembali di berbagai halaman, sehingga mengurangi duplikasi kode dan memudahkan maintenance. Demikian pula, TransaksiService menyediakan method-method yang dapat digunakan oleh berbagai bagian aplikasi untuk melakukan operasi CRUD pada transaksi, sehingga logika bisnis tidak perlu ditulis ulang di setiap tempat yang membutuhkannya.

Aspek **error handling** dan **exception management** merupakan bagian penting dari rekayasa perangkat lunak yang diterapkan dalam aplikasi ini. Setiap operasi yang berpotensi gagal, seperti komunikasi dengan database atau API, dibungkus dengan try-catch block untuk menangani exception yang mungkin terjadi. Aplikasi juga menggunakan structured logging melalui logger package untuk mencatat setiap operasi penting, error, dan warning yang terjadi selama aplikasi berjalan. Logging ini sangat membantu dalam proses debugging dan monitoring aplikasi, terutama ketika aplikasi sudah di-deploy ke production environment. Error handling yang baik memastikan bahwa aplikasi dapat memberikan feedback yang jelas kepada pengguna ketika terjadi kesalahan, dan dapat melakukan recovery atau fallback mechanism ketika operasi tertentu gagal.

Prinsip **scalability** dan **performance optimization** juga diterapkan dalam desain aplikasi. Aplikasi dirancang untuk dapat menangani pertumbuhan data dan pengguna tanpa memerlukan perubahan arsitektur yang signifikan. Penggunaan SQLite untuk local database memungkinkan aplikasi untuk bekerja secara offline dan mengurangi beban pada server, sementara Supabase sebagai backend service menyediakan skalabilitas yang diperlukan untuk menangani pertumbuhan data dan pengguna. Implementasi lazy loading dan pagination pada data transaksi memastikan bahwa aplikasi tetap responsif bahkan ketika jumlah data sangat besar. Optimasi performa juga dilakukan melalui penggunaan FutureBuilder yang hanya melakukan rebuild pada bagian UI yang relevan ketika data berubah, sehingga mengurangi konsumsi resource dan meningkatkan responsivitas aplikasi.

Aspek **security** dalam rekayasa perangkat lunak diterapkan melalui berbagai mekanisme keamanan. Aplikasi menggunakan Supabase Authentication untuk manajemen autentikasi dan otorisasi pengguna, dimana setiap request ke backend divalidasi melalui JWT token. Row Level Security (RLS) policies pada Supabase memastikan bahwa setiap pengguna hanya dapat mengakses data yang sesuai dengan peran dan izinnya. Environment variables digunakan untuk menyimpan sensitive information seperti API keys, sehingga informasi sensitif tidak ter-hardcode dalam source code. Implementasi security yang baik memastikan bahwa aplikasi dapat melindungi data pengguna dan mencegah akses yang tidak sah ke sistem.

Teori rekayasa perangkat lunak yang diterapkan dalam aplikasi ini diperoleh melalui pembelajaran formal pada mata kuliah Rekayasa Perangkat Lunak, serta pengalaman praktis dalam mengimplementasikan berbagai prinsip tersebut dalam proyek nyata. Pengalaman dalam mengembangkan aplikasi Laundry POS System ini memberikan pemahaman yang mendalam tentang bagaimana teori-teori rekayasa perangkat lunak dapat diterapkan dalam konteks pengembangan aplikasi yang kompleks, serta bagaimana berbagai prinsip dan metodologi dapat bekerja bersama untuk menghasilkan perangkat lunak yang berkualitas tinggi dan dapat diandalkan.
